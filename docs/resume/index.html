About Me

I fell in love with purely functional programming in my late teens and went deep — pattern matching, sum types, and Hindley-Milner type inference became second nature. Though not a professional compiler engineer, I’ve spent years building parsers, modeling data structures, and transforming ASTs. This background shaped my approach to solving abstract, technically difficult problems from first principles.

In many ways, I’ve taken a nontraditional path — spending most of my twenties pursuing independent technical work. It taught me how to solve hard problems (though not always how to make money from them). Today, I’m especially strong in systems involving Rust and Swift, particularly in areas that demand deep control, like custom TextKit2 rendering or domains where off-the-shelf tools — including AI — fall short.

Core Competencies

  • Rust and Swift systems programming — CLI tools, cross-platform automation, high-performance backends
  • Custom compilers, parsers, and other analysis tools
  • Text layout/rendering with TextKit2 and CoreText
  • Static site compilers, AI-assisted copywriting pipelines, DOM transformation engines
  • Reverse engineering deeply nested JSON APIs (e.g., Google Maps, obfuscated payloads)

Selected Projects

  • Static Site Compiler: Built a custom site compiler in Rust to replace CMS systems — with macro-style HTML transforms, AI-assisted content generation, and SEO-first architecture.
  • TextKit2 Engine: Developed a custom SwiftUI-compatible text rendering engine for iOS/macOS that goes beyond UIKit’s capabilities.
  • Google Maps Reverse Engineering: Manually dissected and modeled internal JSON payloads from Google Maps (`tbm=map`) for lead generation and business intelligence pipelines.
